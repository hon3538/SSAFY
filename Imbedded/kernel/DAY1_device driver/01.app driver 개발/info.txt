1. 맨처음, 어떤 파일을 만들어야 하며, 어디에 저장되는가?
-> 디파이스 파일 : /dev/

2. 디바이스 파일에 대한 각종 api는 어떤 파일에 작성되는가?
-> 디바이스 드라이버 : system call에 대한 각종 작업(함수)을 등록한다

3. 디바이스 드라이버를 빌드한 후, 모듈을 커널에 장착할 땐 어떤 명령어를 쓰는가?
-> insmod

4. 어떤 파일이 /dev/nobrand 디바이스 파일에 접근해 시스템콜을 일으키는가?
-> app

5. app에서 시스템콜이 일어나면, 무슨 파일에 기록된 대로 행동하는가?
-> 디바이스 드라이버

6. 모듈을 커널의 메모리에서 할당 해제할 땐 어떤 명령어를 사용하는가?
-> rmmod

7. rmmod 이후, app을 실행시키면 정상작동 하는가?
-> X, 디바이스 파일에 접근은 된다. open()에 명령을 수행할 .ko가 없다.

8. 가상머신을 재부팅하면 수동 생성한 디바이스 파일은 존재하는가?
-> X, 그래서 매일 수업할 때마다 sudo mknod / sudo chmod 해야한다.
